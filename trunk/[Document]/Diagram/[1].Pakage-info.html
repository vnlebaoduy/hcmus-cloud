An attribute view provides a read-only or updatable view of the non-opaque values, or metadata, associated with objects in a file system. The com.newatlanta.repackaged.java.nio.file.attribute.FileAttributeView interface is extended by several other interfaces that that views to specific sets of file attributes. FileAttributeViews are selected by invoking the com.newatlanta.repackaged.java.nio.file.FileRef#getFileAttributeView method with a type-token to identify the required view. Views can also be identified by name. The com.newatlanta.repackaged.java.nio.file.attribute.FileStoreAttributeView interface provides access to file store attributes. A FileStoreAttributeView of a given type is obtained by invoking the com.newatlanta.repackaged.java.nio.file.FileStore#getFileStoreAttributeView method.

The com.newatlanta.repackaged.java.nio.file.attribute.BasicFileAttributeView class defines methods to read and update a basic set of file attributes that are common to many file systems.

The com.newatlanta.repackaged.java.nio.file.attribute.PosixFileAttributeView interface extends BasicFileAttributeView by defining methods to access the file attributes commonly used by file systems and operating systems that implement the Portable Operating System Interface (POSIX) family of standards.

The com.newatlanta.repackaged.java.nio.file.attribute.DosFileAttributeView class extends BasicFileAttributeView by defining methods to access the legacy "DOS" file attributes supported on file systems such as File Allocation Tabl (FAT), commonly used in consumer devices.

The com.newatlanta.repackaged.java.nio.file.attribute.AclFileAttributeView class defines methods to read and write the Access Control List (ACL) file attribute. The ACL model used by this file attribute view is based on the model defined by RFC 3530: Network File System (NFS) version 4 Protocol.

The com.newatlanta.repackaged.java.nio.file.attribute.FileStoreSpaceAttributeView class defines methods to read file system space usage related attributes of a file system.

The com.newatlanta.repackaged.java.nio.file.attribute.Attributes utility class defines static methods to access file or file system attribute using the above attribute views.

In addition to attribute views, this package also defines classes and interfaces that are used when accessing attributes:

  • The com.newatlanta.repackaged.java.nio.file.attribute.UserPrincipal and com.newatlanta.repackaged.java.nio.file.attribute.GroupPrincipal interfaces represent an identity or group identity.

  • The com.newatlanta.repackaged.java.nio.file.attribute.UserPrincipalLookupService interface defines methods to lookup user or group principals.

  • The com.newatlanta.repackaged.java.nio.file.attribute.FileAttribute interface represents the value of an attribute for cases where the attribute value is required to be set atomically when creating an object in the file system.
